JS中重要的一环 —— 继承

Seb 2023/7/12 prototype

既然要聊继承，那我们首先要明白继承是什么，我们要通过继承达到的目标是什么？

继承：让子类（派生类）具有父类（超类，基类）的方法和属性

目标：父类构造函数内的属性应该是私有的，每个子类实例对象只维护自己的属性和方法，子类实例对象需要共享的属性和方法要放到他们的prototype上

# 原型链继承

这是JS最简单也是最重要的继承实现 —— 就是让子类的原型对象指向父类的实例对象，这样就把子类插入到了父类的原型链上

子.__proto__ = 子.prototype, 子.prototype.__proto__ =父.prototype

就相当于说子类的构造函数是由父类构造函数创造的

# 实现

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue"];
}
Super.prototype.getSuperName = function () {
  return this.name;
};
function Sub() {
  this.subName = "I am sub";
}

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

// 实现继承关系，要在声明子类构造函数之后

// 将子类的原型对象指向父类的实例对象
Sub.prototype = new Super();

Sub.prototype.getSubName = function () {
  return this.subName;
};

// 这时原来的原型链被破坏
// Sub.prototype.constructor原来是指向Sub的，现在指向了Super
console.log(Sub.prototype.constructor === Super); // true
// 所以我们应该修复这条原型链
Sub.prototype.constructor = Sub;

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();
console.log(sub1.getSuperName()); // I am super

// test仍然是Sub构造的
console.log(sub1.__proto__ === Sub.prototype);

// 测试现在Sub的实例是否在Super原型链上
console.log(sub1.__proto__.__proto__ === Super.prototype); // true
console.log(sub1 instanceof Super); // true

打印出来 sub1 就是如下的状态，已经实现了对Super的继承

# 优缺点

优点：👍

  • 最简单，易于实现
  • 子类实例继承了子类构造函数中的属性，子类原型上的方法，父类构造函数上的属性，父类原型的方法

缺点：👎

  • 对象实例共享所有继承的属性和方法。传教子类型实例的时候，不能传递参数
  • 继承单一，不能同时继承多个父类
  • 所有子实例都会共享父类构造函数的属性，所以父类构造函数引用类型的属性在一个实例上被修改，另一个实例上的属性也会被修改 ❗️

因为sub2没有color这个属性，所以它会沿着原型链找它的父类构造函数，找到 color，然后 push，同样的，sub1 也是沿着原型链找的父类构造函数上的 color，此时已经被 sub2 改变，所以 sub1 的 color 也被修改了

// 当我们修改sub2的color属性时，sub1也发生了改变
sub2.color.push("black");
console.log(sub1.color);
// ["red", "yellow", "blue", "gray", "black"];

同样的原理，父类原型对象上的属性也是一样的（但是父类 prototype 上的属性就是实现共享的）:point_donw:

sub2.arr.push(4);
console.log(sub1.arr);
// [ 1, 2, 3, 4]

# 构造函数继承

在子类型构造函数的内部调用父类型构造函数 —— 使用 apply() 或 call() 方法将父对象的构造函数绑定在子对象上

# 实现

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue", "gray"];
}
Super.prototype.arr = [1, 2, 3];
Super.prototype.getSuperName = function () {
  return this.name;
};

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

function Sub() {
  Super.call(this);
  this.subName = "I am sub";
}

Sub.prototype.getSubName = function () {
  return this.subName;
};

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();

// 获取不到父类原型对象上的方法
// console.log(sub1.getSuperName());
console.log("sub1: ", sub1);

// test仍然是Sub构造的
console.log(sub1.__proto__ === Sub.prototype);

// 测试现在Sub的实例是否在Super原型链上
// 因为没有继承Super的原型对象，所以返回false
console.log(sub1.__proto__.__proto__ === Super.prototype); // false
console.log(sub1 instanceof Super); // false

# 优缺点

优点：👍

  • 可以传参 —— call(传入参数)
  • 可以继承多个父类构造函数的属性 —— 多个call
  • 没有修改原型链

缺点：👎

  • 只继承了父类构造函数中的属性，没有继承父类原型的属性和方法 —— 没有改变原型链 ❗️

子类每一次实例化时，都是一个新的对象，因为 call 是在子类构造函数中的，每次实例化时都会 call 一下，所以修改父类构造函数对象的属性不会互相影响了

// 当我们修改sub2的color属性时，sub1没有影响
sub2.color.push("black");
console.log(sub1.color);
// ["red", "yellow", "blue", "gray"];

# 组合继承

最经典的继承 —— 原型链继承和构造继承：结合了两种模式的优点 —— 传参和复用

# 实现

其实就是两者合到了一起

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue", "gray"];
}
Super.prototype.arr = [1, 2, 3];
Super.prototype.getSuperName = function () {
  return this.name;
};

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

function Sub() {
  Super.call(this);
  this.subName = "I am sub";
}

Sub.prototype = new Super();
Sub.prototype.constructor = Sub;

Sub.prototype.getSubName = function () {
  return this.subName;
};

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();

// 获取不到父类原型对象上的方法
// console.log(sub1.getSuperName());
console.log("sub1: ", sub1);

// test仍然是Sub构造的
console.log(sub1.__proto__ === Sub.prototype);

// 测试现在Sub的实例是否在Super原型链上
console.log(sub1.__proto__.__proto__ === Super.prototype); // true
console.log(sub1 instanceof Super); // true

// 当我们修改sub2的color属性时，sub1不发生改变
sub2.color.push("black");
console.log(sub1.color);
// ["red", "yellow", "blue", "gray"];

// 父类原型对象上的属性和方法依然共享
sub2.arr.push(4);
console.log(sub1.arr);
// [ 1, 2, 3, 4 ]

# 优缺点

优点：👍

  • 结合了原型链继承和构造函数继承二者的优点
  • 可以传参，也可以复用

缺点：👎

  • 代码稍显臃肿，父类构造函数上的属性被重复挂载到了子类的原型上，而子类根本不会通过这条原型链来访问 —— 直接访问父类构造函数
  • 调用了两次父类构造函数 —— 子类构造函数中 call 了一次，原型链继承一次 —— 内存消耗更大

此时父类 prototype 上的属性和方法仍然是共享的，这是对的，因为 prototype 存在的意义就是实现共享

# 寄生组合式继承

对于组合式继承的进一步优化 —— 通过寄生的方式，在组合继承基础上去掉构造父类实例对象这个一个步骤(Sub.prototype = new Super())，取而代之的是让子类的原型对象指向父类副本的实例，从而实现原型链的继承

# 实现

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue", "gray"];
}
Super.prototype.arr = [1, 2, 3];
Super.prototype.getSuperName = function () {
  return this.name;
};

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

function Sub() {
  Super.call(this);
  this.subName = "I am sub";
}

Sub.prototype = Object.create(Super.prototype);
Sub.prototype.constructor = Sub;

Sub.prototype.getSubName = function () {
  return this.subName;
};

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();

// 获取不到父类原型对象上的方法
// console.log(sub1.getSuperName());
console.log("sub1: ", sub1);

// test仍然是Sub构造的
console.log(sub1.__proto__ === Sub.prototype);

// 测试现在Sub的实例是否在Super原型链上
console.log(sub1.__proto__.__proto__ === Super.prototype); // true
console.log(sub1 instanceof Super); // true

// 当我们修改sub2的color属性时，sub1不发生改变
sub2.color.push("black");
console.log(sub1.color);
// ["red", "yellow", "blue", "gray"];

sub2.arr.push(4);
console.log(sub1.arr);
// [ 1, 2, 3, 4 ]

Object.create在这里就是来实现创建父类实例副本的效果，然后让子类实例对象指向它即可 —— 它的原理就相当于下面这段代码👇

function createObj(o) {
  let fn = function () {};
  fn.prototype = o;
  return new fn();
}

Sub.prototype = createObj(Super.prototype);

# 优缺点

堪称完美，同时具备了上述继承方法的优点，同时还很高效，唯一的缺点就是代码比较复杂

❓ 这里有一个问题，为什么不用Sub.prototype.__proto__ = Super.prototype呢

其实是可以的，但是阮一峰在他的文章里有说过，__proto__属性不是语言本身的属性，而是各大厂商添加的私有属性，尽量不要直接使用__proto__，防止对环境产生依赖

# 其他继承

这些继承要么就是很麻烦或者用了很笨的方法实现了继承，要么就是与上面的方法在本质上重复了，了解一下即可👇

# 实例继承

特点：

此时Sub的实例跟Sub没关系，直接是Super的实例了 —— Sub和Super没关系 —— Sub就是一个封装函数的作用

不支持多继承

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue", "gray"];
}
Super.prototype.arr = [1, 2, 3];
Super.prototype.getSuperName = function () {
  return this.name;
};

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

function Sub() {
  var instance = new Super();
  instance.subName = "I am sub";
  instance.getSubName = function () {
    return this.subName;
  };
  return instance;
}

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();

console.log("sub1: ", sub1);

// sub1此时不是Sub构造的了
console.log(sub1.__proto__ === Sub.prototype); // false

// 测试现在Sub的实例是否在Super原型链上
console.log(sub1.__proto__ === Super.prototype); // true
console.log(sub1 instanceof Super); // true

// 当我们修改sub2的color属性时，sub1也发生了改变
sub2.color.push("black");
console.log(sub1.color);
// ["red", "yellow", "blue", "gray"];

sub2.arr.push(4);
console.log(sub1.arr);
// [ 1, 2, 3, 4 ]

// Sub跟Super没关系
console.log(Sub.prototype.__proto__ === Super.prototype); // false

直接打印sub1也可以看出来👇

# 原型继承

就是比实例继承中间多了一个函数进行封装——有点像手写了一个Object.create

不支持多继承

var sub = function (obj) {
  function F() {}
  F.prototype = obj;
  return new F();
};

const sub1 = sub(new Super());
console.log("sub1: ", sub1);

打印sub1, 此时sub1的父类是F，F的父类是Super，通过这样的方式实现sub继承Super

# 寄生继承

也是在中间又加了一层函数来包装，同时结合了实例继承的方法，Sub这个函数只是一个中间环节，不会存在在子类实例的原型链上，跟父类也没什么关系

不支持多继承

var object = function (obj) {
  function F() {}
  F.prototype = obj;
  return new F();
};

function Sub() {
  var s = object(new Super());
  s.subName = "I am sub";
  s.getSubName = function () {
    return this, subName;
  };
  return s; //
}
const sub1 = new Sub();

console.log("sub1: ", sub1);

# 拷贝继承

和实例继承一样，只不过是在子类中遍历父类的实例 —— 然后绑定到子类实例上

function Super() {
  this.name = "I am super";
  this.color = ["red", "yellow", "blue", "gray"];
}
Super.prototype.arr = [1, 2, 3];
Super.prototype.getSuperName = function () {
  return this.name;
};

// 几种方式实现继承的方式主要是下面这部分的代码发生了变化
// ------------------start--------------------------

function Sub() {
  var instance = new Super();
  for (let prop in instance) {
    Sub.prototype[prop] = instance[prop];
  }
  this.subName = "I am sub";
}

// -------------------end--------------------------

// -------------------test-------------------------
const sub1 = new Sub();
const sub2 = new Sub();

// const sub3 = sub(new Super());

console.log("sub1: ", sub1);

打印sub1，发现原型链上并没有Super，因为我们只是把Super实例上的属性赋值到了Sub的原型对象上，只是复制过来，根本没有碰到原型链，所以这其实有点违背了继承的思想😄
