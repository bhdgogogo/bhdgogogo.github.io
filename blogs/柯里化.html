JS中的函数柯里化

Seb 2023/7/11

# 定义

这也是我第一次看到这个概念，之前从来没有听说过😄

维基百科对柯里化的解释为👇

In mathematics and computer science, currying is the technique of translating the evaluation of a function that takes multiple arguments (or a tuple of arguments) into evaluating a sequence of functions, each with a single argument.

一句话来理解就是把多参数传入的函数拆成的单个参数或部分参数的函数

比如下面这个小例子

function add(a, b) {
    return a + b;
}

// 我们一般这样执行 add 函数
add(1, 2) // 3

// 假设有一个 curry 函数可以做到柯里化
var addCurry = curry(add);

// 转换为可以单个参数调用的函数
addCurry(1)(2) // 3

add(1, 2)和addCurry(1)(2)的执行结果是一样的，只是我们将add这个函数给柯里化了，转换为了一个新的函数addCurry

# 优缺点

优点：👍

  1. 参数复用，抽象
  2. 更好的适用性
  3. 入口单一

缺点：👎

  1. 占内存
  2. 效率低 —— 因为使用了递归
  3. 变量存取慢，访问性很差 —— 因为使用了arguments

# 场景

举一个会使用的场景吧

比如，我们要实现一个日志打印的功能，日志打印，一般输出格式是时间+项目名+信息，如2022-07-29 xxx后台管理系统 mm接口异常，要实现这个功能，分别需要时间、项目名、信息三个参数。接下来我们分别用普通函数的写法和柯里化函数的写法，来实现这一功能

  1. 普通函数
// 参数：date, project, message
const log = (date, project, message) => {
  return `${date} ${project} ${message}`
}

const logMsg = log('2022-07-29', 'xxx后台管理系统', 'mm接口异常');
console.log(logMsg) // 输出 2022-07-29 xxx后台管理系统 mm接口异常
  1. 函数柯里化

通常来说，输出日志，一般当天日期是不变的，同一个项目的项目名也是不变的（不过不同的项目名是变化的），唯有信息是时刻变化，那么我们可以将日期这个参数存储起来，这样就不用每次都传日期；当遇到同一个项目的时候，也可以把项目名这个参数存起来，不用每次都传项目名；接下来调用只需要传信息这个参数即可

const log = (date) => {
  
  return (projectName) => {
    
    return (message) => {

      return `${date} ${projectName} ${message}`

    }
  
  }

}


/* 如果日期、项目名、信息都不同的情况下输出日志 */
// 日期为“2022-07-29”，项目名为“A项目”，输出日志
const logMsg1 = log('2022-07-29')('A项目')('接口报错');
console.log(logMsg1); // 打印 2022-07-29 A项目 接口报错
// 日期为“2022-07-29”，项目名为“A项目”，输出日志
const logMsg2 = log('2022-08-01')('B项目')('接口成功');
console.log(logMsg2); // 打印 2022-08-01 B项目 接口成功


/* 如果日期相同，项目名、信息不同的情况下输出日志 */
const sameDateLog = log('2022-07-29');
// 项目名为“A项目”，输出日志
const logMsg3 = sameDateLog('A项目')('接口异常');
console.log(logMsg3); // 打印 2022-07-29 B项目 接口异常
// 项目名为“B项目”，输出日志
const logMsg4 = sameDateLog('B项目')('接口超时');
console.log(logMsg4); // 打印 2022-07-29 B项目 接口超时


/* 如果日期、项目名相同，信息不同的情况下输出日志 */
const sameDateProjectNameLog = log('2022-07-29')('A项目');
// 输出日志
const logMsg5 = sameDateProjectNameLog('网络异常')
console.log(logMsg5); // 打印 2022-07-29 A项目 网络异常

# 实现你自己的curry函数

// 版本1
function myCurry(fn, args) {
  // 获取fn中参数的个数
  var length = fn.length;
  args = args || [];
  return function () {
    // 拼接得到现在所有参数
    for (let i = 0; i < arguments.length; i++) {
      args.push(arguments[i]);
    }

    // 判断参数个数是否大于传入的fn的参数个数
    if (args.length >= length) {
      return fn.apply(undefined, args);
    } else {
      return myCurry.call(undefined, fn, args);
    }
  };
}

// 版本2

function curryIt(fn) {
  let args = [];
  return function curried(arg) {
    args.push(arg);
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    } else {
      return function (args2) {
        return curried.call(this, args2);
      };
    }
  };
}

function add(a, b, c) {
  return a + b + c;
}

var addCurry = myCurry(add);
console.log(addCurry(1)(2)(3));

这个myCurry函数可以被正常调用或者以偏函数（partial）的方式调用

根据定义，柯里化应该将 sum(a, b, c) 转换为 sum(a)(b)(c)

但 JS 中大多数的柯里化实现都是高级版的：它们使得函数可以被多参数变体调用 —— 比如(1, 2)(3)这样的方式调用，上面☝️的两个代码都无法实现这样的效果

所以下面我们可以优化一下，实现一个高级版的柯里化

# 高级版的Curry

function highCurry(func) {
  return function curried(...args) {
    if (args.length >= func.length) {
      return func.apply(this, args);
    } else {
      return function (...args2) {
        return curried.apply(this, args.concat(args2));
      };
    }
  };
}

我们分析if判断

  • 如果传入的 args 长度与原始函数所定义的（func.length）相同或者更长，那么就要调用原始函数了 —— func.apply
  • 否则，获取一个偏函数：我们目前还没有执行包装的原始函数。取而代之的是，返回另一个包装器函数，它将重新应用 highCurry，将之前传入的参数与包装器接收的参数一起传入 —— 递归

# 总结

柯里化是一种转换，将 f(a,b,c) 转换为 f(a)(b)(c) 的形式进行调用。JS 实现通常都保持该函数可以被正常调用，并且如果传入的参数不足，则返回偏函数

柯里化让我们能够更容易地获取偏函数。比如上面的例子，普通函数 log(date, importance, message) 在被柯里化之后，当我们调用它的时候传入一个参数（如 log(date)）或两个参数（log(date, importance)）时，它会返回偏函数，我们只需要通过偏函数传递参数，就可以得到整个log的执行结果
