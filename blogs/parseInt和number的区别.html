parseInt和Number的区别

Seb 2023/7/10 Number

在我们想把一个东西转为 Number 时，常常会使用parseInt()和Number()这两种方法，这篇文章就来总结一下两种法方法的概念，区别和适用场景

# 定义

我们直接从二者的定义来指出二者的区别

  • parseInt —— 将字符串(String)类型转为整数类型
  • Number —— 把对象(Object)的值转换为数字
  • parseInt得到的结果只能是整数或者NaN
  • Number得到的可以是数字（整数，小数等）或者NaN

# parseInt

# 概念

parseInt(string, radix)

参数 描述
string 必需 —— 要被解析的字符串
radix 可选 —— 表示要解析的数字的基数。该值介于 2 ~ 36 之间

radix分析 👇

  • 如果给的值不为0，且不在 2~36 中，则parseInt永远返回NaN
  • 如果给的值在2~36中，那就作为一个基数来处理string

所以在使用parseInt的时候，尽量要指定一个radix的值

如果没有指定radix，或其值为 0或是undefined，则要根据输入的string的值来分析了👇

  • 如果输入的 string以 "0x"或 "0x"（一个0，后面是小写或大写的X）开头，那么radix被假定为16，字符串的其余部分被当做十六进制数去解析。
  • 如果输入的 string以 "0"（0）开头， radix被假定为8（八进制）或10（十进制）。具体选择哪一个radix取决于浏览器。ECMAScript 5 规定使用 10 (十进制)，但不是所有的浏览器都支持
  • 如果输入的 string 以任何其他值开头， radix 是 10 (十进制)。

直接看例子帮助理解 👇

console.log(parseInt("0xF")); // 15
console.log(parseInt("0XF")); // 15
console.log(parseInt("2")); // 2
console.log(parseInt("20")); // 20
console.log(parseInt("09")); // 9
console.log(parseInt("1111112")); // 1111112

如果radix指定在2~36中 👇

比如radix设置为8，则以八进制来解析，设置为16就以十六进制解析

例如：

console.log(parseInt("F", 16));
console.log(parseInt("17", 8));
console.log(parseInt("15", 10));
console.log(parseInt("15"));
console.log(parseInt("15", undefined));

执行☝️这五条语句，输出的结果都是15

# 逻辑

在处理整数的时候parseInt更常用。parseInt函数在转换字符串时，会忽略字符串最前前面的空格，直到找到第一个非空格字符

  • 如果第一个字符不是数字或者正负号，同时没有指定radix的情况下，parseInt就会返回NaN，用parseInt 转换空字符串会返回NaN
  • 如果第一个字符是数字字符，parseInt 会继续解析第二个字符，直到解析完所有后续字符串或者遇到了一个非数字字符，parseInt在解析小数时是向下取整

直接看下面的例子，可以更清楚的帮你理解parseInt的执行机制

console.log(parseInt("15ffasf51")); // 15
console.log(parseInt("  15  ")); // 15
console.log(parseInt("  15dasda  ")); // 15
console.log(parseInt("da15")); // NaN
console.log(parseInt("/15")); // NaN
console.log(parseInt("+15  ")); // 15
console.log(parseInt("")); // NaN
console.log(parseInt("15.6")); // 15

console.log(parseInt("05.6")); // 5
console.log(parseInt("15.6")); // 15
console.log(parseInt("25.6")); // 25
console.log(parseInt("35.6")); // 35

console.log(parseInt("05.6", 2)); // 0
console.log(parseInt("15.6", 2)); // 1
console.log(parseInt("25.6", 2)); // NaN
console.log(parseInt("35.6", 2)); // NaN

console.log(parseInt("05.6", 8)); // 5
console.log(parseInt("15.6", 8)); // 13
console.log(parseInt("25.6", 8)); // 21
console.log(parseInt("35.6", 8)); // 29
  • 二进制只会判断向下取整后的第一个字符是否是0和1，如果是0则返回0，如果是1则返回1
  • 八进制和十六进制会判断向下取整后的整个数字字符

# Number

Number相对来说更自由一些，因为他只有一个参数，同时没有限制参数的数据类型，所以它内部是默认有一套转换规则的，不用我们来指定

参数转换规则如下

  • 如果是Boolean值，true和false值将分别被转换为 1 和 0
  • 如果是数字值，只是简单的传入和返回
  • 如果是null值，返回0
  • 如果是undefined，返回NaN

参数是字符串时：

  • 如果字符串中只包含数字时，将其转换为十进制数值，忽略前导 0
  • 如果字符串中包含有效浮点格式，将其转换为对应的浮点数字，忽略前导 0
  • 如果字符串中包含有效的十六进制格式，如“0xf”，将其转换为相同大小的十进制数值
  • 如果字符串为空，将其转换为 0
  • 如果字符串中包含除上述格式之外的字符，则将其转换为NaN

如果是对象，则调用对象的valueOf方法，然后依照前面的规则转换返回的值。如果转换的结果是NaN，则调用对象的toString方法，然后再依照前面的规则转换返回的字符串值

💪

console.log(Number("test")); // NaN
console.log(Number("")); // 0
console.log(Number(" ")); // 0
console.log(Number("01010101")); // 1010101
console.log(Number(123)); // 123
console.log(Number(123.123)); // 123.123
console.log(Number(false)); // 0
console.log(Number(null)); // 0
console.log(Number(undefined)); // NaN

值得我们注意的是，Number(null)的结果很特殊，是0，这让我们想到了typeof null的结果也很特殊，是object

Number特别的地方还在于一个地方 —— BigInt

MDN上是这样描述的

Number() is the only case where a BigInt can be converted to a number without throwing, because it's very explicit.

意思就是我们可以利用Number()来进行BigInt类型的转换，它不会抛出异常:

但是当BigInt的值很大时，仍会丢失一些精度

+1n; // TypeError: Cannot convert a BigInt value to a number
0 + 1n; // TypeError: Cannot mix BigInt and other types, use explicit conversions

let a = Number(1n);
console.log("a: ", a);
// a: 1

console.log(BigInt(Number(2n ** 54n + 1n)) === 2n ** 54n + 1n);
// false

console.log(Number(2n ** 54n + 1n));
// 18014398509481984
console.log(BigInt(Number(2n ** 54n + 1n)));
// 18014398509481984n
