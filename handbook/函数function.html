# 概念

JS里的函数是被设计为执行特定任务的代码块，它会在某代码调用它时被执行。下面记录一下 JS 中函数的一些基本概念和使用。

# 函数声明

使用function声明函数，并指定函数名 —— function后就是函数名

function test() {
  console.log("我是test函数");
}

// 函数名为test

// 调用test函数
test(); // 我是test函数

# 函数表达式

使用function声明函数，但不在function后指定函数名，此时是匿名函数，将该匿名函数赋予一个变量，通过该变量访问该函数，整个代码就是函数表达式

var test = function () {
  console.log("我是test函数");
};

// 通过变量名test调用函数

test(); // 我是test函数

# 匿名函数

使用function声明函数，但不在function后指定函数名，此时是匿名函数，匿名函数属于函数表达式。

匿名函数有很多作用，将其赋予一个变量 —— 创建函数，赋予一个事件 —— 成为事件处理程序或创建闭包等等。

function test() {
  console.log("我是test函数");
}

// 函数名为test

// 调用test函数
test(); // 我是test函数

# 对比

函数表达式与函数声明的区别

1️⃣ 函数声明可在当前作用域下提前调用执行，函数表达式需等执行到该函数后，才能执行，不可提前调用。

test1();
function test1() {
  console.log("我是test函数");
}
// 我是test函数

test2();
var test2 = function () {
  console.log("我是test函数");
};
// 报错 test2 is not a function

2️⃣ 函数表达式可以直接在后边加括号 —— 实现立即执行

var test = (function () {
  console.log("我是test函数");
})();
// 我是test函数

# 立即执行函数

立即执行函数(IIFE) —— (function(){})() 可以看出很像函数表达式的调用，但为什么要加括号呢？

如果不加括号

function (){
    console.log("我是test函数");
}();
// 报错：Function statements require a function name

此时()前边的函数是匿名函数，因为没有赋值给变量，所以此时不是函数表达式，因此在后边写()是无法运行的 —— 相当于没有函数名，不知道要执行哪个函数

一个语句不能以一个函数表达式开头，因为官方语法假定以单词 function 开头的语句是一个 function 语句 《javascript 精粹》114 页

也就是说 JS 引擎认为以 function 开头的是函数声明语句，不能是函数调用语句。

立即执行函数执行的就是前边的function，所以由var test = function() {}()能够运行 —— 可以理解为将函数声明转化为函数表达式 —— 相当于test()

解决问题的最简单办法就是不让 function 出现在语句的最前面

测试一下在匿名函数前加上其他的操作符

+(function () {
  console.log("1");
})();
// 1

-(function () {
  console.log("2");
})();
// 2

!(function () {
  console.log("3");
})();
// 3

(function () {
  console.log("4");
})();
// 4

由此可见，在前方加特殊的运算符 （这也就是立即执行函数前边为什么要加 “;” ）——可以将函数声明转化为函数表达式，而之所以最后采用括号，是因为括号相对其他运算符会更安全，可以减少不必要的麻烦

使用立即执行函数

(function(a, b){
    console.log(a + b);
})(1, 2)
// 3

(function(){}())这样写的好处是在内部定义的变量不会跟外部的变量有冲突，达到保护内部变量的作用。
