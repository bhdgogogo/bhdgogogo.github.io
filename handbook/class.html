# 介绍

ES6中的class关键字，帮助我们更清晰的实现了js中类的概念，这篇文章就来总结一下class的用法

# 使用

我们使用了class，就是来声明一个类的，而要注意的是，类里面的代码默认就是严格模式

类对象中最重要的一个部分 —— constructor （构造函数）👇

class Person {
  constructor(name, age, gender) {
    this.name = name;
    this.age = age;
    this.gender = gender;
  }
}

构造函数会在我们new Person()的时候执行，证明我们定义了一个Person类的实例对象

如果我们在类中用函数属性省略的写法来定义方法的话，这个方法不会添加到类对象上，而是会添加到该类的原型对象上 —— Person.prototype上

不用省略写法声明的方法，才会被添加到Person类上

class Person {
  constructor(name, age, gender) {
    this.name = name;
    this.age = age;
    this.gender = gender;
  }
  play() {
    console.log("我在玩");
  }
}

const p1 = new Person("smeb", 18, "男");
console.log("p1: ", p1);
console.log("Person.prototype: ", Person.prototype);

打印实例对象可以看到它身上没有该方法，打印Person.prototype可以看到play方法在它身上，当我们调用play时，发现实例对象上并没有该方法，那么就会沿着Person的原型链去找，找到之后，调用该方法

# 封装 —— 安全性

封装就是为了变量私有化 —— 保持数据的安全性

在属性前加上"#"，就可以将属性私有化，外部是无法访问到该属性，这些属性只能在类的内部访问到

class Person {
  #name;
  #age;
  #gender;

  constructor(name, age, gender) {
    this.#name = name;
    this.#age = age;
    this.#gender = gender;
  }
}

console.log(Person.gender);
// undefined
console.log(Person.gender);
const p1 = new Person("smeb");
console.log(p1.name);
// undefined
console.log(p1.sayHello());
// smeb

同时可以看到，该类的实例对象也无法直接访问 —— 使用get和set

在类内部添加get和set方法，来对属性进行读取和修改操作 —— 实现数据的监听，知道是谁在访问数据

class Person {
  #name;
  #age;
  #gender;
  constructor(name, age, gender) {
    this.#name = name;
    this.#age = age;
    this.#gender = gender;
  }

  // getter方法，用来读取属性
  getName() {
    return this.#name;
  }

  // setter方法，用来设置属性
  setName(name) {
    this.#name = name;
  }
}

console.log(p1.getName());
p1.setName("rookie");
console.log(p1.getName());
// smeb
// rookie

# 多态 —— 灵活性

一句话概括多态就是 —— 如果一个东西走路像鸭子，叫起来像鸭子，那么它就是鸭子

调用某个函数，无需制定类型，只要对象满足某些条件即可

class Person {
  constructor(name) {
    this.name = name;
  }
}
class Dog {
  constructor(name) {
    this.name = name;
  }
}
class Test {
  name = "smeb";
}

const dog = new Dog("皮皮");
const person = new Person("小东");
const test = new Test("smeb");
function sayHello(obj) {
  // if(obj instanceof Person){
  console.log("Hello," + obj.name);
  // }
}

sayHello(dog);
sayHello(test);
/*     
    Hello, 皮皮;
    Hello, 小东;
    Hello, smeb; 
  */

# 继承 —— 扩展性

class实现继承利用了另一个关键词 —— extends

当一个类继承另一个类时，你可以理解为将另一个类中的代码复制到了当前类中

class Animal {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(this.name + "在叫~");
  }
}

class Dog extends Animal {}

const dog = new Dog("皮皮");
console.log(dog.name);
dog.sayHello();

// 皮皮
// 皮皮在叫~

如果子类想配置新的属性或新的方法，不应该去修改父类的内容，而应该在子类中进行设置

# 1️⃣ 重写构造函数

有两个关键点，重写构造函数时，在子类的constructor中，要先执行super()，也就是要先执行父类的构造函数，因为没有父类就没有子类

同时要把父类构造函数用到的参数通过 super()传入过去，然后再写子类的构造逻辑

class Animal {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(this.name + "在叫~");
  }
  getAge() {
    console.log("我" + this.age + "岁了!");
  }
}

class Dog extends Animal {
  constructor(name, age) {
    super(name);
    this.age = age;
  }
}

const dog = new Dog("皮皮", 3);
console.log(dog.name);
console.log(dog.age);
dog.sayHello();
dog.getAge();

子类重写了构造函数之后，添加的新属性只在子类上，不会添加到父类上 ☝️

# 2️⃣ 重新父类方法

在子类中，可以通过创建同名方法来重写父类的方法

我们在子类中可以用super来访问父类，可以理解为super就是一个父类对象的虚拟实例 super()就是执行父类的构造函数，super.sayHello()就是执行父类对象的该方法

class Animal {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    console.log(this.name + "在叫~");
  }
  getAge() {
    console.log("我" + this.age + "岁了!");
  }
}

class Dog extends Animal {
  constructor(name, age) {
    super(name);
    this.age = age;
  }
  sayHello() {
    super.sayHello();
    console.log(this.name + "不叫了~");
  }
}

const dog = new Dog("皮皮", 3);
dog.sayHello();

这里的继承方法是基于class的继承，js中还有其他实现继承的方式 —— 利用函数，之后再总结一篇js里的继承，这里就不过多介绍了
