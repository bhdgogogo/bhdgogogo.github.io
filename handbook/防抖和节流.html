防抖节流

Seb 2023/7/3 setTimeout

# 区别

上来先总结一下 👇

防抖是将短时间内的多次执行变为最后一次执行，比如说在 1s 内点击了 10 次按钮，而我们设置的时间是 1s，则生效的只有最后一次点击 —— 不管点击多少次，必须在 1s 内没点击的情况下，才会执行回调函数，如果一直点击，就不会执行回调 —— 注重结果

节流是将短时间内的多次执行变为每隔一段时间执行，比如我们在 1s 内点击了 10 次按钮，则回调函数每 1s 执行一次 —— 我们如果一直点击保持点击，那么回调函数就会每隔 1s 执行一次 —— 注重过程

# 防抖

当触发事件时，必须等到最后一次触发事件达到了我们设置的时间，才会执行回调，如果还没有到，那就重新计时 —— 之前触发的事件都不会执行回调，只有满足要求的情况下，最后一次触发事件才会执行回调

App.vue

<template>
  <div>
    <button id="test" @click="add(), cnt++">点击我测试</button>
    <h2>点击了{{ cnt }}次</h2>
  </div>
</template>

var debounce = function (fn, wait) {
  let timer = null;

  return function () {
    if (timer) {
      clearTimeout(timer);
      timer = null;
    }

    timer = setTimeout(() => {
      fn(this, arguments);
    }, wait);
  };
};

function handle() {
  console.log(Math.random());
}
const cnt = ref(0);

const add = throttle(handle, 1000);

# 节流

当持续触发事件时，保证在一定时间内只调用一次事件处理函数，意思就是说，假设一个用户一直触发这个函数，且每次触发小于既定值（比如我们在 1s 内疯狂触发，其实是不执行回调的，只会到我们疯狂点击的时间持续到了 1s 的时候才会执行回调），函数节流会每隔这个时间调用一次

# 利用时间戳

第一次操作时会立即执行一次回调函数，此后如果还继续有操作，则每隔 1s 执行一次，如果当前这次操作与上次触发回调的间隔不够 1s，就不会触发回调

var throttle = function (fn, delay) {
  // 利用时间戳
  // 第一次操作时会立即执行回调
  let prev = Date.now();

  return function () {
    var context = this;
    var args = arguments;
    var now = Date.now();
    if (now - prev >= delay) {
      fn(context, args);
      prev = Date.now();
    }
  };
};
function handle() {
  console.log(Math.random());
}
const cnt = ref(0);

const add = throttle(handle, 1000);

不能保证最后一次触发会执行一次回调

# 利用定时器

第一次操作不会立即执行回调函数，而是会在 1s 后执行回调，我们在这 1s 内进行的操作，是不会执行回调的，比如我疯狂点击，是不会影响 1s 后执行回调这个事实的，而且这个回调是第一次触发调用的 —— 相当于是一种延迟显示的效果

var throttle = function (fn, delay) {
  // 利用定时器
  // 第一次执行时不会立即执行回调
  let timer = null;

  return function () {
    if (!timer) {
      timer = setTimeout(() => {
        fn(...arguments);
        // 每次执行fn之后，要设置timer为null，否则只会执行一次setTimeout
        // 触发事件就没有效果了
        timer = null;
      }, delay);
    }
  };
};
function handle() {
  console.log(Math.random());
}
const cnt = ref(0);

const add = throttle(handle, 1000);

不能保证第一次触发立即执行一次回调

# 优化版

为更符合现实需求场景，要实现第一次触发马上执行回调，同时最后一次触发也会执行一次回调 👇

var throttle = function (fn, delay) {
  // 优化版
  var timer = null;
  var startTime = Date.now(); //设置开始时间
  return function () {
    var curTime = Date.now();
    var remaining = delay - (curTime - startTime); //剩余时间

    var context = this;
    var args = arguments;

    // 保证多次点击只有一个timer
    clearTimeout(timer);

    if (remaining <= 0) {
      // 第一次触发立即执行
      fn.apply(context, args);
      startTime = Date.now();
    } else {
      // 还没到间隔
      // 在1s内点击多次 —— 保证最后一次触发也会执行一次回调
      timer = setTimeout(fn, remaining); //取消当前计数器并计算新的remaining
    }
  };
};

function handle() {
  console.log(Math.random());
}
const cnt = ref(0);

const add = throttle(handle, 1000);

这样就可以实现我们的需求了~
