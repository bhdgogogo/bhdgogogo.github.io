prototype 和 _proto_ 的区别

seb 2023/5/31 prototype

proto（隐式原型）与 prototype（显式原型）

# 📄 概念

# 1️⃣ prototype

prototype —— 显式原型 —— explicit prototype property

每一个函数在创建之后都会有一个prototype的属性，这个属性指向函数端的原型对象

❗️ 重点： 指向一个对象，而且是函数的原型对象

xxx.prototype，原型对象长什么样呢？，我们输出一下看看

function Foo() {
  this.x = 1;
}

console.log("Foo.prototype: ", Foo.prototype);

它就长上图这个样子，它就是 Foo 的原型对象 —— Foo.prototype，它首先是个对象，同时有一个constructor属性，指回该构造函数Foo，还有一个[[Prototype]]属性，就是它的__proto__，指向该原型对象的构造者的原型对象，对象的构造者都是Object，所以[[Prototype]]就是Object.prototype

你可以用xxx.prototype.__proto__读取到[[Prototype]]，ES6之后，你也可以使用Object.getPrototypeOf(xxx.prototype)读取[[Prototype]]，它与__proto__是等价的

注意：prototype 是构造函数独有的属性

通过Function.prototype.bind方法构造出来的函数是个例外，它没有prototype属性。

# 2️⃣ proto

__protp__ —— 隐式原型 —— implicit prototype link

__protp__用来读取或设置当前对象构造者（构造函数）的原型对象（prototype），JS 中任意对象都有一个内置属性[[prototype]]

Function 的__proto__指向其构造函数 Function.prototype，也就是 Function 的原型对象

Object作为一个构造函数(是一个函数对象!!函数对象!!),所以他的**__proto__**指向 Function.prototype

Function.prototype的__proto__指向其构造函数Object的prototype

Object.prototype 的__proto__值为null —— 到尽头了

显式原型属性和隐式原型属性都指向一个原型对象，只有一个原型对象

对象的__prototype__属性通常与其构造函数的prototype属性相互对应

对象身上一定有__proto__属性
构造函数身上一定有prototype方法

# 作用

  • 显示原型： 用来实现基于原型的继承，还有属性的共享
  • 隐式原型 ： 构成原型链，同样用于实现基于原型的继承。当我们访问obj这个对象中的属性时，如果在obj中找不到，就沿着proto依次查找。

# ⭐️ 原型链

_proto_的指向根据ECMA中的定义 'to the value of its constructor’s "prototype" ' ---指向创建这个对象的函数的显式原型。

所以关键点还是在于找到创建这个对象的构造函数

JS中对象创建的方式，一眼看过去似乎有三种方式：

  • 对象字面量的方式
  • new的方式
  • ES5中的Object.create()

但其实本质上都是通过 new 来创建的

对于原型链以及proto与prototype之间的关系，可以参照下图来理解

关于上图，以下几点需要特别注意：

### :one: 对象

在 JS 里，万物皆对象。方法(Function)是对象，方法的原型(Function.prototype)也是对象。因此，它们都会具有对象共有的特点。

即：对象具有属性__proto__，可称为隐式原型，一个对象的隐式原型指向构造该对象的构造函数的原型，这保证了实例能够访问在构造函数原型中定义的属性和方法

### :two: 函数方法(`Function`)

函数方法这个特殊的对象，除了和其他对象一样有上述_proto_属性之外，还有自己特有的属性——原型属性（prototype）

这个属性是一个指针，指向一个对象 👇

这个对象的用途就是包含所有实例共享的属性和方法（我们把这个对象叫做原型对象）

原型对象也有一个属性，叫做constructor，这个属性包含了一个指针，指回原构造函数

### :three: 构造函数首先是个函数

构造函数Foo()的原型属性Foo.prototype指向了它的原型对象，在原型对象里有共有的方法，所有构造函数声明的实例（这里是f1，f2）都可以共享这个方法。

### :four: 原型对象`Foo.prototype`

Foo.prototype保存着实例共享的方法，有一个指针constructor指回构造函数。

### :five: 构造函数的实例

f1和f2是Foo这个函数对象的两个实例，这两个对象也有属性__proto__，指向构造函数的原型对象，这样对象的实例就可以访问到原型对象的所有方法了

### :six: 构造函数也是个对象 —— 函数对象

构造函数Foo()是个对象，它也有__proto__属性，指向谁呢？ 👇

指向它的构造函数的原型对象呗。函数的构造函数不就是Function嘛，因此这里的__proto__指向了Function.prototype。

其实除了Foo()，Function(), Object()也是一样的道理

原型对象也是对象，它的__proto__属性，同理，指向它的构造函数的原型对象呗。这里是Object.prototype.

最后，Object.prototype的__proto__属性指向null。

## 简单总结
  1. 对象有属性__proto__,指向该对象的构造函数的原型对象
  2. 方法有属性prototype，也有__proto__，prototype指向该方法的原型对象，__proto__指向该方法的构造函数的原型对象
