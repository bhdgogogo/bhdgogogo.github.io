使用Object.defineProperty()

seb 2023/5/17 Object

# 概念

MDN上的解释：

The Object.defineProperty() static method defines a new property directly on an object, or modifies an existing property on an object, and returns the object.

Object.defineProperty() —— 静态方法 —— 直接在对象上定义新属性，或修改对象上的现有属性，并返回该对象。

# 💪分析

let test = {};

console.log("pre: ", test);
// pre: {}

pre: Object.defineProperty(test, "name", {
  value: "test",
});

console.log("cur: ", test); // cur:  {}

上述代码是在node环境中运行的，发现执行完defineProperty后，console.log("cur: ", test)打印不出来添加后属性的对象，难道是没有添加进去？

其实是添加进去属性了，只是没有显示 👇

因为 node 环境下的console.log(obj)遍历对象默认是使用的for...in，而通过defineProperty添加的属性，默认是不可迭代的，所以打印不出来添加后的结果

我们放到浏览器（V8）环境中运行上述代码，就可以正常显示结果（可能两者遍历的底层原理不同吧）

但是如果遍历对象的key时，在浏览器环境也是不会显示的，这时就要设置enumerable为true了

# 配置对象

Object.defineProperty(test, "name", {
  value: "test", // 设置属性默认值
  writable: false, // 设置属性是否能够修改
  enumerable: false, // 设置属性是否可以枚举，是否设置为可枚举属性
  configurable: false, // 设置属性是否可以删除或编辑
  get() {}, // 获取属性的值
  set() {}, // 设置属性的值
});

值得注意的是，在使用Object.defineProperty定义属性的时候，configurable、enumerable、writable的值默认都为false。
如果是直接在对象上定义属性，configurable、enumerable、writable的值默认为true

(obj.myName = ‘bian’)，这时 name 属性就时可枚举的，可以被修改的，可以被删除或编辑的
