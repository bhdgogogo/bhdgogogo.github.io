toString()理解

seb 2023/5/11 stringObject

# 方法描述

☀️简单概括如下：

toString() —— 返回对象的字符串表示形式。

@paramradix -指定将数值转换为字符串的基数。只能用于Number类型

MDN上的描述为：

  • 除了 null 和undefined之外，其他基本类型（数值，Boolean，字符串，对象）都有toString() 方法，它返回对应值的字符串表现（并不修改原变量）
  • 每个对象都有一个方法
  • 当该对象被表示为一个文本值时，或者一个对象以预期的字符串方式引用时自动调用。
  • 默认情况下，toString()方法被每个Object对象继承，如果此方法在自定义对象中未被覆盖，则toString()返回“[object type]”，其中 type 是对象的类型

# 默认的toString()方法

默认情况下，toString()方法被每个Object对象继承，如果此方法在自定义对象中未被覆盖，则toString()返回 “[object type]”，其中 type 是对象的类型。

let a = 10;
console.log(a.toString())       // "10"，返回对象的字符串表示形式

console.log(a.toString(2))       // "1010"，返回a的二进制表示字符串

({}).toString()  				// [object Object]
// 注意不能直接写成 {}.toString()，因为 {}会被当做代码块而不是空对象

let b = true;
console.log(b.toString());      // "true"
let c = "wujingyue";
console.log(c.toString());      // "wujingyue"
let d = [1, 2, 3, 4];
console.log(d.toString());      // "1,2,3,4"
let e = Symbol();
console.log(e.toString());      // "Symbol()"
let f = undefined;
console.log(f.toString());      // 报错 “Cannot read property 'toString' of undefined”
let g = null;
console.log(g.toString());      // 报错 “Cannot read property 'toString' of null”

# 覆盖toString方法

我们可以自定义一个方法，来覆盖默认的toString()，默认的toString()方法不能传入参数，且必须返回一个字符串，而我们自定义的toString()方法可以传入自定义的参数，同时可以返回我们想要返回的数据


// 声明一个对象
function Player(name, team){
	this.name = name;
	this.team = team;
}

let player = new Player("sicca", "dys");
console.log(player);  	// Player { name: 'sicca', team: 'dys' }
console.log(player.toString());		// [object Object]
console.log(Object.prototype.toString.call(player));	// [object Object]

⬆️此时没有重写toString()，player调用toString()方法，会继承Object的toString()方法，返回[object Object]

Player.prototype.toString = function(){
	return `我是${this.name}，来自${this.team}`;
}
console.log(player.toString());		// 我是sicca，来自dys
console.log(Object.prototype.toString.call(player));	// [object Object]

⬆️通过.prototype.toString重写Player对象的toString方法，此时player再调用toString()方法，就会执行我们的自定义函数

# 💪 补充

细心的朋友会看到，我在两个地方都额外添加了一个输出 —— Object.prototype.toString.call()，在我们重写对象的toString()时，player.toString()的结果发生了改变，但Object.prototype.toString.call()的值没有改变，一直是[object Object]

这提醒了我们，在判断对象类型时，toString()方法是不可靠的，因为我们可能会重写对象的toString()函数（Array, function等类型作为Object的实例，默认都重写了toString()方法）

因此，想要得到对象的具体类型时，应该使用Object上原型的toString()方法（未被重写），也就是Object.prototype.toString.call()
