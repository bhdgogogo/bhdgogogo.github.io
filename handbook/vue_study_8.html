# 双重数据绑定

这里我们将表单项的 value 属性和变量 text 做了绑定
当 value 发生变化时，text 变量会随之变化（单向绑定）

let text = ref(" ")
<input @input="(event) => (text = event.target.value)" />

当 value 或 text 任意一个发生变化，另一个也会随之变化（双向绑定）

<input
  type="text"
  @input="(event) => (text = event.target.value)"
  :value="text"
/>

双重数据绑定的作用就是让 vue 可以获取到该值（数据），从而我们修改该值（数据）时，不用去找 html 元素，只需要找我们双重绑定的那个变量即可，这让我们的开发更加灵活

# v-model

v-model = "text" —— v-model:value的简写 —— 因为v-model默认收集的就是value值

就等价于

@input="(event) => (text = event.target.value)" :value="text"

可以自己制定 true 和 false 的值，比如设置 true 为是，设置 false 为否 true-value = "是"

false-value = "否"

v-model只能应用与表单类元素（输入类元素）上，比如input, select等

v-model 修饰符

  • .lazy 使用 change 来处理数据
  • .trim 去除前后的空格
  • .number 将数据转换为数值

# 自定义事件(emits)

# 什么时候用？

当我们想要调用父组件的方法时，而利用props传数据是自上向下的 —— 父给子传，所以我们通过自定义事件的方式（emits）来实现回传的效果，从而完成父子组件之间的通信 —— 子调用父的方法

# 怎么用？

父组件写函数，通过@的形式发送给其他组件

@del="xxx"

子组件通过$emit("xxx")来调用父组件传过来的方法

但是此时父组件是不知道我们子组件有什么自定义事件的（缺少关联性），就是不知道@后的的内容 所以我们一般用如下的方式来使用 emits

// 定义一个自定义事件xxx
const emits = defineEmits(["del"]);

// 然后在父组件就可以获取到该组件的自定义事件，然后给该自定义事件绑定函数
@del="xxx"

// 子组件就可以通过emits来调用父组件的函数方法了
emits("xxx")

子组件声明一个自定义事件，通过emits("xxx")来调用父组件中的方法

# 杂

submit 默认会进行页面的跳转 —— submit.prevent 可以取消默认跳转行为

event （函数）是 vue 给我们的自动传进来的

event.target 表示的是触发事件的元素，事件是谁触发的就返回谁

自己的事情自己做，一个vue文件就只干一个工作，在该文件的数据就在该文件中处理，其他文件想要读取数据可以，但是要修改这些数据就要调用该文件的方法。

有很多场景我们进行字符串拼接时会同时使用单引号和双引号，这个时候要遵循一个原则：
要成对嵌套使用，外层用单引号里面就用双引号，外层用双引号里面就用单引号

要传递一整个字符串对象时，外层要用双引号，里面用单引号，变量拼接处肯定使用的是双引号，就可以传递成功

  • 加上花括号：只导入模块中的指定变量或函数
  • 不加花括号： 导入整个模块

如果导入的模块没有默认导出（即没有export default语句），则必须使用花括号来指定导入的变量或函数。如果导入的模块有默认导出，则可以使用不加花括号的语法来导入整个模块对象。

tbody外不能用div来装，如果tbody外层有div，那就跟表头tr不对应了，样式会乱

事件访问需要在事件前加上on，而且通过透传访问事件时也要加on，并且要用驼峰命名

数据和数据的操作方法应该放在一个组件中

把响应式对象转化为普通对象

  1. Object.assign —— 浅复制 —— 可以用来
  2. 解构展开 —— {... xxx}
