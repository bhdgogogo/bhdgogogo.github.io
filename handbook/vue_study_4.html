# 选项式 api 和组合式 api

选项式api

不同的选项配置不同的内容，data就配置data，methonds就配置methonds

组合式api

钩子函数—— setup

可以通过这个函数向外部暴露组件的配置

对象的形式编写不方便 => setup函数编写

在组合式api中直接声明的变量，就是一个普通的变量，不是响应式的，修改这些属性时，不会在视图中产生效果

使用reactive import { reactive } from "vue" const a = reactive({xxx}) —— 对该对象进行代理，并产生一个响应式的对象赋值给a

<script setup>
  //
  可以直接编写所有类型的代码，声明变量，申明函数，不用再写export，data这些，直接像编写正常代码一样编写即可
</script>

shallowReactive——浅层代理，只将根位置的元素变成响应式数据

reactive —— 深层代理，所有元素都会变成响应式数据，不管它在第几层

# ref

reactive只能操作对象 —— ref ref的原理： ref在生成响应式代理时，它是将值包装为了一个对象（RefImpl） 0 --> {value:0} 访问ref对象时，必须通过 对象.value 来访问其中的值 在模板中，ref对象会被自动解包

vue给我们提供了一个语法糖 => $，使得ref对象在script标签中也可以自动解包

但$是Beta的，需要在vite插件中（vite.config.js）做一些配置 —— reactivityTransform:true $ref()，实际开发中谨慎使用

ref对象在模板中可以自动解包（前提是，ref对象必须是顶层对象，如果顶层对象不是 ref，内层对象是ref，那调用内层对象不会自动解包，就需要加伤value来显示对应值了）

# 模板语法

除了组件中的变量外，vue 也给我们提供了一些全局对象来供我们访问

比如：Date, Math, RegExp, JSON... （vue文档可以查到）

同时，可以往应用中添加全局变量 —— app.config.globalProperties（一般也用不到） 使用插值(双大括号)，只能使用表达式

# v-bind

当我们需要为标签动态设置属性时，就需要使用v-bind指令
v-bind可以简写为 :

使用时可以理解为 :属性名=“变量”

需要一下子设置多个属性可以使用v-bind

当我们为一个布尔值设置属性时， 如果值为true，则元素上有该属性（转换后为true，也算true） 如果值为false，则元素没有该属性（转换后为false，也算false） 特殊情况："" 空串，在这里会被当成真值

# 杂

  • 改变量只会影响到变量自己，在js中，无法实现对一个变量的代理
  • 在模板中，通过双大括号（插值）来直接访问到组件实例的属性（组件中声明的变量）
  • 所有全局的配置都是在app（app代表整个应用）中设置的
  • 插值实际上就是在修改元素的 textContent =>（如果内容中含有标签，标签会被转义显示，不会作为标签生效）
  • 空串通常情况下会被转换为false
