for...in和for...of的比较

seb 2023/5/16 ArrayObject

# ⭐️概念

for...in是ES5的标准，该方法遍历的是对象的属性名称(key：键名)。一个Array对象也是一个对象，数组中的每个元素的索引被视为属性名称，所以在使用for...in遍历Array时，拿到的是每个元素索引

一般用于遍历对象自身的和继承的可枚举属性。以及对象从构造函数原型中继承的属性。对于每个不同的属性，语句都会被执行。 for…in 语句以原始插入顺序迭代对象的可枚举属性(只能迭代出可枚举的属性，可枚举属性【js 自定义属性】/不可枚举属性【对象的内置属性，如数组的length 就是一个内置属性，所以for…in遍历不出来】)。 -for…in的原理是 —— Object.keys()：返回给定对象所有可枚举属性的字符串数组

  • 一般不建议使用for...in 遍历数组，因为输出的顺序是不固定的。
  • 如果迭代的对象的变量值是null或者undefined, for...in不执行循环体，建议在使用for...in循环之前，先检查该对象的值是不是null或者undefined

for...of是 ES6 的标准，该方法遍历的是对象属性所对应的值(value：键值)。所以遍历数组时会得到每个元素的值

  • for…of 语句在可迭代对象（包括 Array，Map，Set，String，TypedArray，arguments 对象 等等）上创建一个迭代循环，调用自定义迭代钩子，并为每个不同属性的值执行语句
  • for…of 语句遍历可迭代对象定义要迭代的数据（非自定义属性）
  • for…of 循环可以使用的范围包括数组、Set 和 Map 结构、某些类似数组的对象、Generator 对象，以及字符串。

# 💡测试

# 遍历 Array, Set, Map

let arr = ["A", "B", "C"];
let set = new Set(["A", "B", "C"]);
let map = new Map([
  [1, "A"],
  [2, "B"],
  [3, "C"],
]);

console.log(arr);
console.log(set);
console.log("map:", map);

// 遍历数组
for (let x of arr) {
  console.log(x); // 输出： A B C  —— 数组元素中的值
}

for (let x in arr) {
  console.log(x); // 输出： 0 1 2  —— 数组元素的下标
}

// 遍历Set
for (let x of set) {
  console.log(typeof x);
  console.log(x); // 输出： A B C —— 就是依次输出Set中的元素
}

for (let x in set) {
  console.log("x", x); // 没效果，没有输出 —— 不能使用for...in遍历Set
}

// 遍历Map
for (let x of map) {
  console.log(typeof x);
  console.log(x); // 输出： [ 1, 'A' ] [ 2, 'B' ] [ 3, 'C' ] —— 依次输出Map中的元素
  console.log(x[0] + ":" + x[1]); // 输出： 1:A 2:B 3:C —— 可以获取到map的key和value
}

for (let x in map) {
  console.log("x", x); // 没效果，没有输出 —— 不能使用for...in遍历Map
  // 所以肯定获取不到map的key和value
}

# 遍历对象和数组

  • 遍历对象
let s = { a: 1, b: 2, c: 3 };

for (let prop in s) {
  console.log(prop); //a b c —— 依次输出对象的属性
  console.log(s[prop]); //1 2 3 —— 输出对象对应属性的值
}

for (let prop of s) {
  console.log(prop); //报错： s1 is not iterable
}

let ss = Object.keys(s); // 返回一个表示给定对象参数的所有可枚举属性的字符串数组
console.log(ss); // 输出： [ 'a', 'b', 'c' ]

for (let i of ss) {
  console.log(i); // a b c
  console.log(s[prop]); // 1 2 3
}

for (let i in ss) {
  console.log(i); // 0 1 2
}
  • 遍历数组
let arr = ["a", "b", "c"];
console.log("arr:", arr); // arr: [ 'a', 'b', 'c' ]
arr.hobby = "football"; // 这样添加的属性是可枚举的，但是for...of是遍历不到的
/* Object.defineProperty(arr, "age", {
  value: 18,
});
这样添加的属性是不可枚举的，也就是不能遍历的，for...in和for...of都无法遍历到
*/
console.log("arr:", arr); // arr: [ 'a', 'b', 'c', hobby: 'football' ]

for (let i in arr) {
  console.log(i); //  0 1 2 hobby
  console.log(arr[i]); //  a b c  football
}
for (let i of arr) {
  console.log(i); //  a b c —— 遍历不到添加的 football 值，因为football的下表是hobby，for...of通过next()访问不到该下标，所以遍历不到该值
}

Object.prototype.objCustom = function () {};
Array.prototype.arrCustom = function () {};

console.log("arr:", arr); //  arr: [ 'a', 'b', 'c', hobby: 'football' ]

for (let i in arr) {
  console.log(i); // 0, 1, 2, "hobby", "arrCustom", "objCustom"
}
// hasOwnProperty表示是否有自己的属性。
// 这个方法会查找一个对象是否有某个属性，但是不会去查找它的原型链。
for (let i in arr) {
  if (arr.hasOwnProperty(i)) {
    console.log(i); //0, 1, 2, "hobby"
  }
}

for (let i of arr) {
  console.log(i); // 'a', 'b', 'c'
}

hasOwnProperty 表示是否有自己的属性，该方法会查找一个对象是否有某个属性，但是不会去查找它的原型链

let str = new String();

console.log(str.hasOwnProperty("split")); //  false
console.log(String.prototype.hasOwnProperty("split")); // true

# 📝总结

  • for...of 循环的是可迭代对象的值（value）
  • for...in 循环的是可迭代对象的属性（key）

注意：for…of 循环不能循环普通的对象，因为 Object 对象本身没有内置的 Iterator，它并没有实现这个接口，对普通对象的属性遍历推荐使用 for…in

如果非要使用 for…of 来循环对象，不是不可以，要结合 Object.keys() —— 其实也就是 for...in 的原理进行使用

let s = { a: 1, b: 2, c: 3 };
for (let key of Object.keys(s)) {
  //使用Object.keys()方法获取对象key的数组
  console.log(s[key]);
}

在 for...of 中无法改变数组的值，因为 for..of 的本质是返回一个迭代器，迭代器是通过 next()方法返回值，而不是指向地址，所以在 for...of 迭代中无法改变数组的值

let arr1 = [1, 2, 3];
for (let i in arr1) {
  arr1[i] *= 2;
}
console.log(arr1); // [ 2, 4, 6 ]

let arr2 = [1, 2, 3];
for (let i of arr2) {
  i *= 2;
}
console.log(arr2); // [ 1, 2, 3 ]

在JS 中，没有指针或者引用赋值这种东西，a = b[i]不意味着 a &= b[i]或者 a =& b[i]，对基础类型而言，始终是值的复制，对于对象/数组/函数而言，始终是传引用。 所以 —— a = b[i] 只是值的复制

# 💪 Vue 场景

  • v-for(value,index)-in 与 v-for(value,index)-of 在遍历数组的时候，value 得到的是数组的值，index 是数组的索引值 v-for-in 与 v-for-of 在遍历数组的时候这俩者没有区别
  • v-for(value,index)-in 与 v-for(value,index)-of 在遍历对象的时候，value 是对象的健值，index 是对象的健名，v-for-in 与 v-for-of 遍历对象的时候这俩者也没有区别的

所以在 vue 中，v-for...of 和 v-for...in 在遍历数组和对象时是没有区别的，都可以遍历

可枚举属性和不可枚举属性，forEach 和 map

# for...in补充

使用for-in循环，返回的是所有能够通过对象访问的、可枚举的属性，其中既包括存在于实例中的属性，又包括存在于原型中的属性。就算是屏蔽了原型中不可枚举属性的实例属性也会在for-in循环中返回

看下面这个例子就可以👇

Object.prototype.gender = "男";
var person = {
  name: "smeb",
  age: 12,
};
for (var key in person) {
  console.log(key, person[key]);
}
/* 
  name smeb
  age 12
  gender 女
*/

for...in遍历person这个对象，对象上没有gender这个属性，但是它的原型链上有gender 这个属性，所以我们遍历时也会访问到gender并输出Object.prototype.gender的值

如果你只想访问该对象上的属性，就使用hasOwnProperty来过滤一下 👇

Object.prototype.gender = "男";
var person = {
  name: "smeb",
  age: 12,
};
for (var key in person) {
  if (person.hasOwnProperty(key)) {
    console.log(key, person[key]);
  }
}
/* 
  name smeb
  age 12
*/

如果你只想访问该对象上的可枚举属性，就使用Object.key()👇

Object.prototype.gender = "男";
var person = {
  name: "smeb",
  age: 12,
};
console.log(Object.keys(person));
// [ 'name', 'age' ]

如果你想访问该对象的所有属性，不包括原型链，包括可枚举和不可枚举，就使用Object.getOwnPropertyNames() 👇

Object.prototype.gender = "男";
var person = {
  name: "smeb",
  age: 12,
};
console.log(Object.getOwnPropertyNames(Object.prototype));
/* 
[
  'constructor',
  '__defineGetter__',
  '__defineSetter__',
  'hasOwnProperty',
  '__lookupGetter__',
  '__lookupSetter__',
  'isPrototypeOf',
  'propertyIsEnumerable',
  'toString',
  'valueOf',
  '__proto__',
  'toLocaleString',
  'gender'
]
*/

其中valueOf, toString, hasOwnProperty, propertyIsEnumerable都是不可枚举的属性或方法
