javascript判断对象类型

seb 2023/4/28

js中有多种方式可以判断对象的类型，每种方式都有自己的优缺点。这篇文章中一共总结了 4 种方法：

  1. instanceof
  2. typeof
  3. Object.prototype.toString.call
  4. constructor

# instanceof

# 怎么用?

一种关键字判断方法

obj instanceof Object

左边是对象实例，要判断的对象 右边是构造函数 如果obj继承于Object.prototype，就返回true，否则返回false

这里的继承可以是直接继承，也可以是间接继承，也就是说构造函数只要在某一个实例原型链上，那instanceof的结果就为true

# 缺点

  1. 多窗体中（比如嵌套了iframe），每一个窗体有独立的上下文，在一个iframe中的Array实例，在父窗口中使用instanceof Array的结果是false，因为两者上下文独立，两个Array是相互独立的。（解释一：同源的iframe是可以通过dom访问到的，能够拿到iframe里面的对象，当要处理跨文档的类型判断时，instanceof就会出现问题
  2. 只能判断某一个实例是不是某一个构造函数的实例，不能取到具体的类型。如果我要拿到obj的类型比如Array，instanceof是拿不到的

# typeof

# 怎么用？

typeof 是 js 内置的一个操作符，后面接一个对象的实例或者原始类型

console.log(typeof "123"); // string
console.log(typeof 123); // number
console.log(typeof null); //object
console.log(typeof undefined); // undefined
console.log(typeof new Array()); // object
console.log(typeof (() => "")); // function

typeof null —— 返回的是object

# 缺点

主要用来判断原始类型，如果是对象，那只能返回Object，不能知道该对象类型的构造函数类型，也就是具体的对象类型，比如是数组，字符串，JSON这些

typeof是内置的操作符，js不像c++，没有提供内置操作符重写的能力，所以如果想让typeof [1,2,3]返回’array’是不可能的

# Object.prototype.toString.call

# 怎么用？

js是有类型的概念的，通过Object默认的toString方法 —— Object.prototype.toString()，它返回一个对象的字符串形式，默认情况下返回类型字符串。因此可以用来判断一个值的类型。

var o = {};
o.toString(); // "[object Object]"

上边的代码调用空对象的toString方法，结果会返回一个[object Object] —— 名值对

  • 第一个object是小写，表示的是东西的结构性质，这个东西是个对象
  • 第二个Object是大写，指的是该值的构造函数，表示构造函数被调用

所以上述对象的类型是对象

console.log(Object.prototype.toString.call(new Date())); // [object Date]
console.log(Object.prototype.toString.call(JSON)); // [object JSON]
console.log(Object.prototype.toString.call(Math)); // [object Math]
console.log(Object.prototype.toString.call(new f())); // [object Object]
console.log(Object.prototype.toString.call(f)); // [object Function]

从 JavaScript 1.8.5 开始，Object.prototype.toString.call(null)返回的是[object Null]，Object.prototype.toString.call(undefined)返回的是[object Undefined]

通常，当我们讨论 javascript中的对象时，我们指的“对象对象” 数组就是数组对象，函数就是函数对象 [object，Array] —— [对象，数组] —— 数组对

# 缺点

对于大部分的内置类型，我们都可以通过Object.prototype.toString获取到类型字符串。 但是我们不能通过获取到自定义构造函数的具体的类型，比如我们获取new f的类型字符串，返回的是[object Object]。解决这个问题，一种方式是重写f.prototype.toString，让其返回[object f]。但是当我们的toString另有他用的时候（参考Date的toString），这种方式就行不通了，而且每个新的构造函数都要重写toString，很麻烦。

# constructor

# 怎么用

通过原型对象的constructor属性，可以获取到构造函数，所以我们可以通过构造函数取到构造函数的名字 —— 构造函数的名字也可以代表类型。 在ES6里面提供了函数的name属性，我们可以通过function.name这种方式拿到函数名

function f() {}
console.log(f.name); // f

ES5 及以下可以通过 function.toString 来拿到函数定义的字符串，然后从中取出构造函数的名字

console.log(
  f
    .toString()
    .match(/function(.*)\(.*/)[1]
    .trim()
);

# 缺点

有些函数时没有名字的 —— 匿名函数

var range = function (x, y) {}; // 这个函数是没有名字的

constructor属性时可以被修改的，所以这种方法并不可靠 同时，只能判断它的构造函数，不能得到原型链上的构造函数

# 总结

  • 如果希望判断是不是基础类型的或者对象，或者是函数，可以使用typeof
  • 如果希望区分内置对象和自定义对象，可以使用Object.prototype.toString.call
  • 如果只是想要判断某一个对象是不是某一个构造函数的实例，使用instanceof
  • 如果希望得到自定义对象的具体的类型的字符串，可以使用constructor

每一种方法都有自己的特点，理解了原理之后，可以根据你自己的需要，将几种方式组合起来使用
