正则表达式

Seb 2023/7/5 RegExp

# 介绍

正则表达式不是哪一门语言独有的 —— 它代表了一种规则 —— 符合我们约束的规则

百度百科对其的描述如下 👇

正则表达式，又称规则表达式,（Regular Expression，在代码中常简写为regex、regexp或RE），是一种文本模式，包括普通字符（例如，a到z 之间的字母）和特殊字符（称为"元字符"），是计算机科学的一个概念。正则表达式使用单个字符串来描述、匹配一系列匹配某个句法规则的字符串，通常被用来检索、替换那些符合某个模式（规则）的文本。

用我自己的话概括就是，正则表达式是一种人为可以设置的规则，用来判断字符串是否符合我们的要求

# 使用

# 构造函数

在JS中，正则表达式是一个对象，我们可以直接new一个RegExp对象

const req = new RegExp()

RegExp()接受两个参数

  1. 第一个参数为字符串或一个RegExp对象 —— 匹配字符串 —— 必须
  2. 第二个参数表示optional flags —— 匹配模式 —— 可选 （比如g表示全局，i 表示不区分大小写，m表示多行）

利用test方法来判断传入的字符串是否符合要求，我们第一个参数设置了“a”，表示传入的字符串为"a"就返回true，否则返回false

当我们配置第二个参数“i”时，表示忽略大小写，所以此时要判断的字符串是“A”或“a”都会返回 true，否则返回 false

const req1 = new RegExp("a");
const req2 = new RegExp("a", "i");
let a = "A";

console.log(req1.test(a));
// false
console.log(req2.test(a));
// true

# 直接使用

const req = /a/i，这样写的 req 与上面的const req2 = new RegExp("a", "i")效果是一样的

req.exec() —— 用来获取字符串中符合正则表达式的内容

# 常用的

  1. 在正则表达式中大部分字符都可以直接写

  2. | 在正则表达式中表示或

  3. [] 表示或（字符集）

    [a-z] 任意的小写字母 [A-Z] 任意的大写字母 [a-zA-Z] 任意的字母 [0-9]任意数字

  4. [^] 表示除了 ——> [^x] 表示除了x

  5. . 表示除了换行(\n\r)外的任意字符

  6. 在正则表达式中使用\作为转义字符

  7. 其他的字符集

    \w 任意的单词字符 [A-Za-z0-9]

    \W 除了单词字符 [^A-Za-z0-9]

    \d 任意数字 [0-9]

    \D 除了数字 [^0-9]

    \s 空格

    \S 除了空格

  8. 开头和结尾

  • ^ 表示字符串的开头
  • $ 表示字符串的结尾
  1. 常用修饰符

    i —— ignore, 不区分大小写
    g ——global 全局匹配
    m —— multi line 多行匹配 s —— 特殊字符圆点 . 中包含换行符 \n —— 默认是不包含的

# 💭 一个例子

检验输入的字符串是否为合格的url

# 分析

  • 开始符: ^
  • 协议名： http(s)或ftp, rtsp😕/ 表示 ——> ((https|http|ftp|rtsp|mms)?:\/\/) 没有可以，但是有的话只能有其中的一个，所以用?修饰
  • 域名部分：主机名可以使用"-"符号，所以两种情况都要判断，包含"-"或不包含"-" 包含"-" ——> [A-Za-z0-9]+-[A-Za-z0-9]+ 字符组都是不止一个 wdw-32dw 不包含"-" ——> [a-zA-Z0-9]+ 字符组也是不止一个所以要在最后加上+ 比如www 然后加上点，最后要设置+ 因为www.www.www.这种情况也是存在的
  • 顶级域名： com,cn 这些域名的长度都在 2-6 位 ——> [a-zA-z]{2,6}
  • 端口 比如:8080 不止一个数字 可以没有 有的话只能有一个 /d用来匹配数字 ——> (:\d+)?
  • 请求路径： 比如/login/item 可以有多个 可以没有请求路径 但是这个部分有的话只能有一个 ——> (\/.*)？
  • query 传参和哈希值 比如?age=1 可以有多个 可以不传参 但是这个部分只能有一个 ——> (\?.*)(#.*)？
  • 结束符： $

# 代码实现

const _isUrl = (url) => {
  // 补全代码
  let reg =
    /^((https|http|ftp|rtsp|mms)?:\/\/)(([a-zA-Z0-9]+-[a-zA-Z0-9]+|[a-zA-Z0-9]+)\.)+([a-zA-Z]{2,6})(:\d+)?(\/.*)?(\?.*)?(#.*)?$/;
  return reg.test(url);
};

function test() {
  const judge1 = _isUrl("http://.com:80?name=?&age=1");
  const judge2 = _isUrl("https://a.b.c.com:8080/get?name=?&age=1");
  const result = !judge1 && judge2;
  return result;
}

console.log(test());
// true
