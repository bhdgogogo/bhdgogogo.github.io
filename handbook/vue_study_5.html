# scoped

如果直接通过style标签写样式，此时编写的样式是全局样式，会影响到所有的组件

添加scoped属性之后，样式将成为局部样式，只对当前组件生效

如何实现的？

  • 当我们在组件中使用scoped样式时，vue会自动为组件中的所有元素生成一个随机的属性，形如：data-v-7a7a37b1,生成后，所有的选择器都会在最后添加一个[data-v-7a7a37b1]
  • h1 -> h1[data-v-7a7a37b1]—— 属性选择器
  • .box1 -> .box1[data-v-7a7a37b1]

注意：

  • 随机生成的属性，除了会添加到当前组件内的所有元素上
  • 也会添加到当前组件引入的其他组件的根元素上（前提引入的组件是单根组件）
  • 这样设计是为了，可以通过父组件来为子组件设置一些样式

:deep() —— 特殊场景会用到，可以改变当前组件中所有该元素的样式 :global()—— 全局选择器

# module

css模块

  • 自动的对模块中的类名进行 hash 化，来确保类名的唯一性

  • 在模板中可以通过 $style.类名 使用，比如$style.box1

  • 也可以通过module的属性值来指定变量名，比如设置module="c"，就通过c.box1调用

    注意，通过这种方式来声明的样式，在使用class调用时，要使用:class="xxxx"，因为此时样式成为了变量

# props

子组件中的数据通常不会在子组件中直接定义，因为这样会导致数据和视图发生耦合，显示数据的组件就负责显示数据，传数据的组件就负责传数据

子组件中的数据通常会在创建组件实例时确定

父组件可以通过 props 来将数据传递给子组件

使用props

  1. 先在子组件中定义props
  2. 父组件通过props定义的属性来把数据传给子组件

注意：父组件传递给子组件的props都是只读的，无法修改 —— 单项数据流 —— 确保数据的安全性，降低项目代码的复杂度
即使可以修改，我们也尽量不要在子组件中去修改父组件的数据
如果非得要改，具体方法后边再讲（自定义事件）

props的配置

通过对象的方式来给props赋值

const props = defineProps({
  count: Number,
  obj: Object,
  isCheck: Boolean,
  maxLength: {
    type: String, // 类型
    required: true, // 帮助检查有没有传该属性，写不写都行
    default: "哈哈", //  默认值
    validator(value) {
      // 校验
      return value !== "嘻嘻";
    },
  },
});

# 杂

在vue3中定义组件的方式有两种

  1. 单根组件 一个div然后里边写其他的元素
  2. 多根组件 直接写元素 不在外边套一个div

  • 类和内联样式也是可以通过变量的方式来绑定的

  • 不要追求一步到位，就一点一点的做

  • 结构，表现，行为

  • 设计思想是，各组件之间尽量减少互相影响，降低组件之间的耦合

  • 每一层外部都尽量设置一个容器

  • 渐变不是颜色，是图片

  • 不用引入的属性或者函数，称作为宏指令，比如defineProps，就称作一个宏

    图片在垂直放入容器中时，图片底部与容器中间有一条缝 —— 修改vertical-align 的值，只要不是baseline，问题即可解决
