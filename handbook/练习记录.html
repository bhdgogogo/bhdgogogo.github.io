# Meal 组件

抽象成两个组件

  1. MealBody组件 —— 整个菜单
  2. MealComponent组件 —— 每个单独的汉堡

MealBody组件要注意高度，留出上下两个部分的高度，否则会挡住菜单界面

MealComponent组件结构

  1. 左边汉堡图片
  2. 右边汉堡信息info + 选购组件（Counter组件）

左右为flex结构，右边信息为上下两部分结构

# Counter 组件

两个组件都设置 display: flex 后，两个组件从左右排列变成了上下排列 原因：两个组件变成了块元素，块元素默认是上下排列的 解决： 给父组件也设置 display: flex

  • justify-content 属性定义了项目在主轴（x 轴）上的对齐方式。
  • align-items 属性定义项目在交叉轴（y 轴）上的对齐方式。
// 可以同时设置两个类的属性，一般用于设置多个具有相同样式的类
.aa,
.bb {
  display: flex; // flex: 设置该元素为块级元素
  margin: 0 2rem; //上下边距为0，左右边距为2rem
  padding: 0 1rem 0 2rem // 上右下左
;
}

给store设置action来绑定添加汉堡和删除汉堡事件 —— 在state中设置count来记录 —— 通过action来操作对应汉堡count的增和减

需要MealComponent组件把当前汉堡属性传给Counter，从而获得当前汉堡的state，如果直接从store读state的话，只能获取所有汉堡，而无法获取到当前汉堡

Counter组价要实现的需求如下：

  1. 获取购物车所有商品 —— 用一个数组来记录count > 0的汉堡
  2. 计算总数量 —— count —— 购物车右上角小图标
  3. 计算总价格 —— count * price —— 购物车总价格

# MealFilter 组件

搜索食物列表的搜索框组件

通过在 state 中设置一个 keyword 属性来实现，通过 v-model 绑定到 input 上，根据输入内容来对 data 进行 filter，得到搜索后的结果

如果在 template 外手动结构了数据内容，则该数据内容是一次性的，也就是第一次解构的是什么内容，后边再进行过滤或刷新当前组件内容，该数据内容仍然是第一次解构的数据

解决： 在模板中通过 props 访问

indexOf —— 不重复（没有该元素）表示 -1, 重复（有该元素）表示 1

举例：参考 (opens new window)

判断两个数组中元素是否重复，不重复返回值-1，重复则返回元素首次出现的位置。

  1. 创建带有重复的数组和一个空数组
var arr1 = [1, 13, 2, 3, 4, 6, 2, 8, 5, 8, 10, 2, 5, 13, 11];
var arr2 = [];
  1. 通过 for 循环遍历，if 判断两个数组中元素是否重复，不重复返回值-1，通过 push方法将 arr1 中不重复元素添加到 arr2 中

# CartBar 组件

下方购物车条 position:fixed —— 固定定位

结构:

  • 购物车 —— CartBar —— 底部购物车的一个整个样式
  • 购物袋 —— 包括右上角商品数量小图标
  • 选择内容 —— 未选中商品 —— 选中商品显示选中商品的总价格
  • 结算 —— 结算按钮 —— 最左边

assets 文件夹放的是我们项目的资源 —— 比如 css 等 —— 一般是不会改变的图片 —— 直接要用的东西 如果是之后会改变的资源 —— 数据图片这些 —— 就放到 public 文件夹里

绝对定位是相对于最近 position 不为 static 的父级元素来定位的。参考 (opens new window)

flex 不好操作时可以使用绝对定位来设置元素位置

# CartDetails 组件

slot —— 我们希望父组件的功能不变，然后指定父组件里要显示的内容

以前父组件的内容是写死的，在哪儿引入父组件，里边的内容都是固定的 —— slot解决

问题：数据是在当前组件中获取的 —— 只能显示固定数据 —— 耦合 —— 不能复用 —— 其他组价调用时只能得到固定的数据，无法根据需求来显示数据 解决： 把获取数据的工作抽象出去，当前组件通过属性的配置来得到要显示的数据，其他组件在使用时，也通过属性来设置要显示的数据，要显示啥就传啥

把mealBody中的数据给抽象出去

  • 问题：筛选框失效
  • 解决：把调用了筛选方法的数据传过去

待解决问题：

  1. props能传类，但是无法调用传过来的类的属性和方法？是不是props通常只会传数据（Aarry）
  2. 购物车商品手动删除空之后，显示“暂无该商品信息” —— 是filter查询不到的内容 —— 应该显示购物车内容为空的信息 —— "暂无选购商品信息" —— 已解决（给mealBody添加属性，控制是在购物车内显示还是在主列表显示）

在一个组件中可以修改另一个组件的样式，直接通过css定义.xxx来修改，前提是要修改的组件最外层的div的类是xxx，不是最外层是无法修改的

# Dialog 组件

也是继承于Mask组件 —— 但要注意与CartDetail组件的区别 因为是Mask的子组件，所以可以调用Mask的hide方法，不必再单独写一个方法来控制Dialog的隐藏和显示

注意调用的是Mask的emits事件，但是不影响其他使用Mask组件的地方，只是进行了dialog的显示和隐藏 —— 为什么？—— 每个组件是独立的

# Checkout 组件

点击去结算之后显示该组件，内容与购物车内容一致，但细节上有区别

  1. 首先是一个大背景框，充满整个屏幕，点击去结算按钮显示，点击close符号关闭
  2. detail内容部分又分成三部分，header，购物车内容list，footer
  3. list分成照片（上），info（下）两部分(info上面放汉堡名字，下面放Counter和数量)
  4. footer参考Cartbar组价的结算部分内容即可

发现与Cartbar有很多相同代码，优化 —— 把下方的结算部分给抽象成一个组件 —— Pay.vue

# 杂

vw 视口宽度
图的单位和 rem 的单位对应起来了 —— calc(100vw / 750) —— 750 是设计图的宽度 —— 把 100vw 平均分为了 750 份 —— 1rem = 100 / 750 vw vh 视口高度

图片下间隙（有多余空白）问题的解决方法

参考 (opens new window)

  1. 设置 img 样式 vertical-align —— 只要不是 baseline 即可
  2. 设置 img 为 display:block —— 把图片转块
  3. 给图片的父元素设置 font-size 或 line-height 为 0

vertical-align 的原因分析

参考 (opens new window)

可以理解为图片后又一个空白节点，图片的 vertical-align：baseline（默认），图片需要后虚拟的空白节点 baseline 对齐，所以将父级容器的下边沿向下撑开了。
