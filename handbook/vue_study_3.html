# index.js 里发生了什么

createApp(App)：将根组件关联到应用上，会返回一个应用实例

应用实例是注册一个可供组件在应用内使用的“全局变量”，可以理解为是我们整个的一个项目

返回的应用实例可以用于注册全局组件

注意：应用上只能关联一个根组件 一旦挂载到一个div上，那原本div中所有的内容都会被覆盖

写在html中的代码会被html优先解析成小写的格式，如果在index.js写成了大写，那就会报错，因为找不到该组件

组件实例是一个Proxy()对象——代理对象——对该对象进行改造

组件是可以重复使用的，也就是创建多个应用实例（但在实际开发中基本上不会这样做）

data会返回一个对象作为返回值，vue会对该对象进行代理，从而将其转换为响应式数据—— return 的内容才是响应式数据

但直接向组件实例中添加的属性不会被vue所代理，这些属性的值也就不是响应式数据

可以修改这些属性的值，但是修改这些属性的值时，不会重新渲染页面

# 响应式原理——代理

为对象创建一个代理

const proxy = new Proxy();

代理规则（行为）通过配置一个对象来设置，get()设置读取数据时的行为，它的返回值就是最终读取到的值，指定get后， 在通过代理读取对象属性时，就会调用get方法来获取值，而set()会在通过代理修改对象时调用

为什么要设置get和set? 设置了之后，读取和修改属性的行为就变成了在调用方法，那调用方法就变得有意义了 可以在get中，返回读到的值的前面添加一些操作，可以在set中，修改值之后做一些其他的操作

trigger()触发所有的使用该值的位置进行更新——响应式数据

只有通过代理对象去修改属性时，才有响应式数据的效果

# data

一句话概述：data返回的对象最终会被vue所代理

想用的属性，就算暂时不用，也先写上去，可以先赋值为null，后边要用到的时候就可以直接读和改了，就不用往里边加了，在开发中，读取和修改是容易实现的，而如果想要往里边加属性的话，可能会产生未知的问题

# methods

methods是一个对象，在对象中可以添加多个方法

这些方法最终会挂载到组件实例上，然后就可以通过组件实例调用了

methods中的this就是组件实例

组件就相当于一个类，data 里的是属性，methods里的是方法

methods在使用时可加括号也可以不加括号，但一般我们约定，调用methods时，我们不加括号，比如 @click = "change"

# computed

用来指定计算属性，是通过方法来返回的，属性显示的时候有逻辑就用computed info就是计算属性

info 和 methonds计算的区别

  1. info只在其依赖的数据发生变化时才会重新执行（也就是会对数据进行缓存）
  2. 而在methonds中的方法，在组件每次重新渲染时都会执行

在getter中（也就是info），尽量不要写修改属性的代码，只写根据属性来判断，然后输出。只有getter时可以简写计算属性

计算属性可以设置setter，但是不建议这么做，没有set时，计算属性就是只读的，我们尽量保持computed是只读的，这有利于代码的维护

# 杂

在标签中插值的时候，也就是双大括号里边，只能写表达式

有返回值的语句就叫表达式

理解原理——搞清楚这件事的来龙去脉

在组件中注册子组件时要注意大小写！
